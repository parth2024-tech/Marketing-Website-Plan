 this is website idea Marketing Website Plan A dark, futuristic multi-page marketing site for a Windows background app that predicts laptop hardware problems before they happen. Working name: Sentinel (easily renamed later). Both "Download for Windows" (placeholder link) and "Join the waitlist" CTAs, with the waitlist storing emails via Lovable Cloud. Pages (separate routes for SSR + SEO) / — Home: Hero with animated diagnostic/pulse visual, value prop ("Knows your laptop is sick weeks before you do"), 3 key differentiators, social-proof-style metric strip, dual CTAs, condensed feature highlights, waitlist section, footer. /features — Features: Grid of the 8 unique capabilities (predictive, learns your machine, habit coaching, plain English, weekly report, personalized tips, correlation detection, gets smarter). /how-it-works — How it works: 3–4 step flow (Install silently → Learns your normal → Detects drift & correlations → Tells you in plain English). Includes a sample "weekly health letter" mock and a sample alert mock. /what-it-monitors — What it monitors: Visual grid of the 12 subsystems (Battery, SSD, CPU, RAM, GPU, thermals, fans, power delivery, startup health, memory pressure, network stability, drivers) with one-line plain-English descriptions. /faq — FAQ: Privacy (local-only), resource usage, supported Windows versions, why predictive vs reactive, etc. /waitlist — Waitlist: Dedicated email signup with confirmation state. Shared Header (logo + nav + waitlist CTA) and Footer rendered in __root.tsx. Visual design (dark / techy / futuristic) Tokens added to src/styles.css (oklch): Background: near-black with subtle blue tint Foreground: soft off-white Primary: electric cyan (signal/health) Accent: violet (prediction/AI) Warning: amber (habit alerts) Gradients: --gradient-pulse (cyan→violet), --gradient-surface (panel sheen) Glows: --shadow-glow-primary, --shadow-glow-accent Custom radii, monospaced accents for "metric chip" components Effects: Animated radial pulse / EKG-style line behind hero (CSS + SVG, no heavy libs) Subtle grid background, glow on primary buttons, soft card borders Lucide icons, Inter for body, a geometric display font (e.g. Space Grotesk via Google Fonts) for headlines Fully responsive, accessible contrast, light/dark tokens defined (defaults to dark) CTAs Primary: Download for Windows — button links to # placeholder (you can swap the URL later) Secondary: Join the waitlist — opens /waitlist or scrolls to inline form on home Backend (Lovable Cloud) Enable Lovable Cloud to power the waitlist: waitlist_signups table: id, email (unique), source (page), created_at RLS: public INSERT only (no select), so emails can be submitted but not read from the client Submission via a createServerFn that validates the email with Zod and inserts via the authed client Success/error toast with sonner SEO Each route sets its own head() with unique title, description, og:title, og:description. Root __root.tsx updated with global brand defaults and a sensible favicon-ready setup.
